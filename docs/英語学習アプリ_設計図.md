# 英語学習アプリ 設計図（ハンドオフ仕様書）

> このドキュメントは、現行の家族向け英語学習アプリ (`HayashiFamilyEnglish.html`) を
> ベースに、**仲間が同じ思想で別アプリをゼロから作れる**ように要件をまとめたものです。
> 後半では **NOT A HOTEL 版**（ラグジュアリー・ホスピタリティ向け）への寄せ方を
> 具体的に提案します。
>
> 想定読者: エンジニア / プロダクト担当 / コンテンツ制作者
> 前提知識: HTML・JavaScript・React の基礎

---

## 0. このアプリの「正体」を一言で

**「1 つの HTML ファイルだけで完結する、オフライン対応の英語自習アプリ」。**

- サーバーもビルドもいらない。`.html` を 1 つ開けば動く。
- データ（単語・例文・長文）はすべてファイル内に直書き。
- 学習の進捗は端末の localStorage に保存。インターネットが切れても使える。
- （任意で）Supabase に進捗を保存して端末間・チーム間で同期。
- （任意で）Claude API を入れると AI 翻訳・AI 添削が動く。

この「軽さ」が思想の核です。**学習者が毎日ストレスなく開ける**ことを最優先にしています。

---

## 1. 設計思想（これだけは外さない 5 原則）

1. **オフラインファースト** — 電車の中でも、Wi-Fi がなくても、開けば必ず動く。
2. **ゼロ摩擦** — ログイン画面・ローディング・チュートリアルで止めない。プロフィールを選んだ瞬間に学習が始まる。
3. **レベル別に最適化** — 同じアプリを、初心者も上級者も使える。出題は自動でレベル調整。
4. **毎日続く仕掛け** — 「今日やること」を自動で提示し、迷わせない。
5. **美しさは実用** — 落ち着いた配色・上質なフォント・気持ちいいアニメーション。「開きたくなる」ことが継続率を決める。

## 2. 技術スタック

| 領域 | 採用技術 | 理由 |
|---|---|---|
| UI | React 18（CDN 読み込み、**JSX 不使用**） | ビルド不要。`React.createElement` を `e` にエイリアスして手書き |
| スタイル | CSS-in-JS（インライン style オブジェクト） | 外部 CSS 不要。コンポーネントと同居 |
| 永続化（端末） | localStorage | サーバー不要。オフラインで完結 |
| 同期（任意） | Supabase（`@supabase/supabase-js` を CDN） | 単一テーブルの JSONB に丸ごと保存するだけ |
| AI（任意） | Anthropic Claude API（ブラウザから直接 fetch） | 翻訳・添削・チャット |
| 音声 | Web Speech API（`speechSynthesis`） | 追加コストゼロ。端末内蔵の音声を使う |
| 配信 | GitHub Pages（静的ホスティング） | `.html` を置くだけ |
| オフライン | Service Worker + manifest（PWA 化） | ホーム画面に追加してアプリのように使える |

> **重要**: フレームワークもバンドラ（webpack/vite 等）も使っていません。
> `index.html` を 1 枚置けば本番稼働します。これは「壊れにくさ」と「引き継ぎやすさ」のための意図的な選択です。

## 3. ファイル構成

```
/
├── index.html              ← アプリ本体（実体は単一巨大 HTML、~1.6MB / 約 29,000 行）
├── sw.js                   ← Service Worker（オフラインキャッシュ）
├── manifest.json           ← PWA マニフェスト（アイコン・名前・テーマ色）
├── icon-180.png / 192 / 512 / icon.svg  ← アプリアイコン各サイズ
└── serve.js                ← ローカル確認用の簡易サーバー（node serve.js）
```

HTML 1 枚の中は、ざっくり以下の順で並んでいます：

```
<head>  … フォント読み込み / PWA メタ / Service Worker 登録
<body>
  <script> React/ReactDOM/Supabase を CDN から読み込み </script>
  <script>  ← ここが本体（数万行）
    1. 定数・デザイントークン（配色 C、フォント）
    2. 学習者プロフィール（PROFILES）
    3. 機能タイル定義（FEATURES）
    4. 学習コンテンツ（VOCAB / SENTENCES / SHADOWING_QA …）← データの塊
    5. データ層（db オブジェクト = localStorage 読み書き + 同期）
    6. 学習エンジン（今日のパス生成 / レベル調整 / 覚えた判定）
    7. 音声（speak / getBestVoice）
    8. AI 連携（Claude API ラッパー）
    9. React コンポーネント群（各画面）
    10. ルートの App コンポーネント + ReactDOM.render
  </script>
```

## 4. データモデル（コンテンツの構造）

すべて JavaScript の配列としてファイル内に直書きします。
現行アプリの規模感（参考値）：

| データ | 役割 | 規模 |
|---|---|---|
| `VOCAB` | 単語＋イディオム。例文を 2 つ持つ | 約 2,000 語 |
| `SENTENCES` | 瞬間英作文用の短文（文法タグ付き） | 約 730 文 |
| `SHADOWING_QA` | シャドーイング用の単文・Q&A・必修フレーズ | 約 410 文 |
| `SHADOWING_PASSAGES` | 長文（スピーチ・読み物） | 約 33 本 |
| `CONVO_PATTERNS` | 「定型構文」。1 構文につき 3 例文 | 144 構文（432 文） |
| `DAILY_PHRASES` | シーン別の日常フレーズ | 80 フレーズ |
| `ENGLISH_LESSONS` | 1 分解説＋5 問クイズ | 44 レッスン |
| `LISTENING_QUIZ` | 聞き取り 4 択 | 70 問 |
| `READING_QUESTIONS` | 長文読解 4 択 | 長文ごとに数問 |

### 4.1 単語（VOCAB）の形

```js
{
  id:'water', en:'water', ja:'水', pos:'noun', level:1, cat:'daily',
  exEn:'I drink water every day.',  exJa:'私は毎日水を飲みます。',
  exEn2:'Could I have some water?',  exJa2:'お水をいただけますか？'   // 第2例文（違う意味/場面）
}
```

- `level` … 1〜5（後述のレベル設計）
- `pos` … 品詞（noun/verb/adj/idiom…）
- `cat` … カテゴリ（daily/family/school/general… 自由に定義）
- **例文を 2 つ持つ**のがポイント。同じ単語でも違う場面・違う意味で 2 回出会わせると定着が段違い。

### 4.2 短文（SENTENCES）の形

```js
{ id:'s001', level:1, grammar:'be動詞', ja:'私は学生です。', en:'I am a student.' }
```

- `grammar` … 文法パターンのラベル（「現在完了」「関係代名詞」など）。出題のグルーピングと表示に使う。

### 4.3 定型構文（CONVO_PATTERNS）の形 ★アレンジの肝

SNS（Instagram / TikTok / YouTube）で頻出する「型」を集めたもの。
**1 つの型につき必ず 3 例文**を持たせ、「同じ型を 3 連続で繰り返すと頭に残る」という学習仮説を実装しています。

```js
{
  id: 'cp001',
  pattern: "I'm having trouble ...ing",   // 型（英語）
  ja_pattern: '〜するのに苦労している',      // 型（日本語）
  category: '基本',
  items: [
    { ja: '漢字を覚えるのに苦労している。', en: "I'm having trouble remembering kanji." },
    { ja: '夜寝つきが悪くて困っている。',   en: "I'm having trouble sleeping at night." },
    { ja: '宿題に集中できなくて困っている。', en: "I'm having trouble focusing on my homework." },
  ],
}
```

- レベルは別テーブル（`CONVO_PATTERN_LEVELS`）で `cp001 → LV2` のように割り当て。
- アプリ起動時に自動で「瞬間英作文」と「シャドーイング」のプールへ流し込み（後述）。

### 4.4 長文（SHADOWING_PASSAGES）の形

```js
{
  id: 'pas_lessons_01', level: 3,
  title: 'English lessons · ① 自己紹介',
  summary: '自己紹介の英語',
  en: "Let me introduce myself. ...",   // 50〜150 words 程度
  ja: '自己紹介をさせてください。…',       // 対訳
}
```

- 学習者本人のための「実際に話すシチュエーション」を題材にすると、当事者性が出て続く。
  （現行では「自己紹介」「家族の紹介」「仕事のこと」「夢のこと」を本人が書いて添削して収録）

## 5. 学習者プロフィール & レベル設計

### 5.1 プロフィール（PROFILES）

```js
{ id:'father', name:'林亮治', subtitle:'英検3級レベル', color:C.brand, role:'learner', level:3 }
```

- 複数の学習者を「カード」で切り替える設計。家族・チームで 1 アプリを共有できる。
- `role: 'learner' | 'viewer'`（viewer は閲覧寄り。応援担当など）
- `level` … その人の基準レベル。出題の初期値になる。
- `forProfileIds` をタイル側に持たせると「この人にだけ出すタイル」も作れる。

### 5.2 レベル（5 段階・英検目安）

| LV | 英検目安 | 中身 |
|---|---|---|
| 1 | 5〜4 級 | 超基本単語・be 動詞・現在形 |
| 2 | 4〜3 級 | 過去形・助動詞・日常会話 |
| 3 | 3〜準2級 | 現在完了・関係詞の入口 |
| 4 | 2 級 | 受動態・分詞・仮定法の入口 |
| 5 | 準1級 | 仮定法・倒置・高度イディオム |

### 5.3 comfort（自動レベル調整）★これも肝

固定レベルだけだと「簡単すぎる/難しすぎる」が起きるので、
**正解率に応じて出題レベルを少数で上下させる `comfort` 値**を track（vocab / composition）ごとに持ちます。

- マスターが進む → comfort が上がる → 1 つ上のレベルが混ざり始める
- 間違いが続く → comfort が下がる → 易しめに戻る
- これにより「ちょうどいい難易度」が自動で維持される。

## 6. 機能一覧（タイル）

ホーム画面は機能タイルのグリッド。`FEATURES` 配列で定義し、プロフィールの role で出し分け。

| タイル | 内容 |
|---|---|
| シャドーイング | 聞いて即真似する音読。単文・Q&A・長文 |
| 単語学習（日→英 / 英→日） | フラッシュカード。例文 2 つ＋発音 |
| 瞬間英作文（日→英） | 日本語を見て英語を組み立てる |
| 間違い復習 | 間違えた問題だけ再出題 |
| 覚えた一覧 | マスター済みの語・文を一覧 |
| 1 分英語解説 | 文法テーマの短い解説＋5 問クイズ |
| AI 英作文テスト | 週次・3 問・AI 採点 |
| リスニング理解 | 音声を聞いて 4 択 |
| リーディング理解 | 長文を読んで 4 択 |
| AI 翻訳 | 日⇄英、写真の文字も |
| 翻訳ノート | 保存した翻訳で復習 |
| レベル診断テスト | 段階式で現在地を測る |
| 定型構文 | 144 構文を 3 つの形式で練習（後述） |

### 6.1 「定型構文」の 3 練習モード ★最重要アレンジ

同じ 432 文を、**3 つの異なる脳の使い方**で回せるようにしたのが特徴。

1. **英作文モード** — 日本語表示 →（自分で考える）→「答えを表示」→ 覚えた/覚えていない を記録
2. **シャドーイングモード** — 英語を 2 回自動再生 → 真似して言う → 自動で次へ
3. **思考タメモード** — 日本語表示 → 数秒カウントダウン（頭の中で英作文）→ ネイティブ音声で答え合わせ

共通仕様：
- **順番 / ランダム** を切り替え可能（ランダムでも「同じ型の 3 例文の固まり」は崩さない）
- 答えのヒントになるので、**型名（"I'm having trouble"）は答えが出るまで隠す**。答えの下に「構文: …」として出す。
- 各カード右上に「× この問題を今後出さない」ボタン（チーム全員に同期）

## 7. 学習エンジン

### 7.1 今日の学習パス（getOrCreateTodaysPath）

毎朝、その人のレベルと進捗から「今日やること」のステップ列を自動生成して保存。
ホームで「次は ◯◯ をやろう」と 1 つだけ光らせ、迷わせない。

- 例: シャドーイング → 単語 → 間違い復習 → 瞬間英作文 → リスニング → リーディング → …
- 曜日条件（週 2 で AI テストを挟む等）や、間違いがある時だけ復習を挟む等の分岐を持つ。
- 全部終わると「完走」演出。その後も「もう一回」を巡回提示して継続を促す。

### 7.2 覚えた / 間違い 判定

- 文・単語ごとに `{ attempts, correct, consecutiveCorrect, lastAt }` を記録。
- **連続 3 回正解でマスター**（= 以後の通常出題から外れる）。
- 間違えると mistakes 配列に積まれ、「間違い復習」で再出題。正解すると外れる。
- 「克服した間違い数」を実績として可視化（モチベーション設計）。

### 7.3 「外した問題」(hiddenItems)

「この例文いらない」を × で外せる。`{ scope: { id: true } }` の形で保持し、
**チーム全員の端末に同期**（後述のクラウド共有行を使う）。設定から一括復元も可能。

## 8. 音声読み上げ（TTS）

`Web Speech API`（`speechSynthesis`）を使用。端末内蔵の音声なので追加コストゼロ・オフライン可。

**重要なノウハウ**: 何もしないと機械的な「棒読み」になる。
端末が持つ高品質ボイスを起動時に検出して優先選択する `getBestVoice()` を必ず入れる。

優先順（実装済み）:
1. Microsoft の Neural オンライン音声（Edge / Windows）
2. Apple の Premium / Enhanced ボイス（iOS / Mac の Ava・Samantha・Daniel 等）
3. Google US/UK English（Android Chrome）
4. それ以外の en-* 音声

> iOS は「設定 → アクセシビリティ → リーダーと読み上げ → 声 → 英語」で
> Enhanced/Premium ボイスを一度ダウンロードすると、以後自動で使われる。

## 9. AI 連携（Claude API・任意）

- 設定画面で各自が API キーを入力（端末に保存）。キーがなければ AI 機能だけ非表示でも動く。
- ブラウザから `https://api.anthropic.com/v1/messages` に直接 fetch
  （`anthropic-dangerous-direct-browser-access: true` ヘッダが必要）。
- 用途:
  - **AI 翻訳**（日⇄英、写真 OCR からの翻訳）
  - **AI 英作文テスト採点**（3 問を採点＋優しく訂正＋ヒント）
  - **別の言い方の提示**（瞬間英作文で「他の自然な言い方」を 2〜3 個）
- システムプロンプトで「温かく」「具体例を 1 つ」「優しく訂正」というトーンを固定。
- モデルは軽量・高速なものを採用（採点・翻訳には十分）。

> セキュリティ注意: ブラウザ直叩きなので API キーは各端末ローカル管理。
> 不特定多数に配る本番では、軽量プロキシ（キーをサーバー側で保持）を挟むのが安全。

## 10. クラウド同期（Supabase・任意）

**思想: localStorage が常に主。クラウドは「バックアップ＆同期」。**

- Supabase に `member_state` という**単一テーブル**だけ用意。
  - `member_id`（text, PK）/ `data`（jsonb）/ `updated_at`（timestamp）
- 学習者ごとに 1 行。`data` に進捗 JSON を丸ごと upsert（デバウンスで 1 セッション 1 回程度に集約）。
- プロフィール選択時にクラウドを pull。**新しい方で上書き（last-write-wins）**。
- オフライン時は pending に積み、オンライン復帰で自動再送。

### チーム共有行（family-shared）

`member_id = 'family'` の特別な行に、**全員で共有したいもの**をまとめる:
- 表示名・アバター・写真
- 「外した問題」(hiddenItems)

これで「誰かが外した例文は全員の端末から消える」が成立。

## 11. オフライン対応 & PWA

- `manifest.json` でアプリ名・アイコン・テーマ色・`display: standalone` を定義 → ホーム画面に追加でアプリ化。
- `sw.js`（Service Worker）:
  - HTML（アプリ本体）は **network-first**（更新を取りに行き、失敗したらキャッシュ）
  - 静的アセット・CDN は **cache-first**
  - API（AI）は**キャッシュしない**
  - `CACHE_VERSION` を上げると古いキャッシュを破棄。
- **更新導線**: アプリ内に `APP_VERSION`（数字）を表示。新 SW を検知したら自動リロードで最新へ。
  - 「ちゃんと新しいのが入ったか」を学習者が数字で確認できることが、サポートコストを劇的に下げる。

## 12. UI / デザイン原則

現行は「上質な飲食店」を思わせる落ち着いたトーン。NOT A HOTEL 版では**ブランドガイドに差し替え**る。

- カラーは定数 `C`（bg / card / text / sub / line / accent / success / danger …）に集約 → 一括変更可能。
- フォント: 表示用セリフ（見出し）/ 本文用セリフ（読みやすさ）/ サンセリフ（UI ラベル・数字）を使い分け。
- 余白・角丸・薄い境界線で「静けさ」を出す。装飾過多にしない。
- アニメーションは控えめ・短め（押した感、進捗バー、完走演出くらい）。

## 13. コンテンツ制作ガイド

仲間がコンテンツを足すときの基準:

- **例文は「その人が実際に言いそう」を最優先**。教科書的な無味乾燥より、当事者の生活・仕事に直結した文。
- 単語は**最低 2 例文**（違う意味 or 違う場面）。
- 長文は **50〜150 words** が音読しやすい。長い話は ①②③ と章分け。
- 名言・格言の類は**入れすぎ注意**。抽象的・比喩的で会話に出てこない英語は、初学者のやる気を削ぎ、定着もしない。入れるなら「短くて誰もが知っているもの」だけ。
- 構文（型）は **1 つにつき 3 例文**をセットで。

## 14. ゼロから作る手順（推奨ステップ）

1. **骨組み**: 単一 HTML に React/ReactDOM を CDN で読み込み、`e = React.createElement`、空の `App` を描画。
2. **デザイントークン**: 配色 `C`・フォントを定義。
3. **データを少量**: 単語 20・短文 20・長文 2 本だけ入れて形を固める。
4. **localStorage 層 `db`**: 読み書き・進捗記録（attempts/correct/mistakes）。
5. **最初の 1 画面**: 単語フラッシュカード（表示→めくる→覚えた/覚えていない）。
6. **プロフィール選択**とレベル概念を入れる。
7. **タイル化**: 機能を増やす（瞬間英作文 → シャドーイング → リスニング …）。
8. **今日の学習パス**で毎日の導線を作る。
9. **TTS**（getBestVoice）を入れて音を出す。
10. **PWA 化**（manifest + Service Worker + APP_VERSION 表示）。
11. **コンテンツを本量に拡充**。
12. （任意）**Supabase 同期** → **Claude API**。
13. GitHub Pages にデプロイ。

> 1〜5 だけで「使えるアプリ」になります。あとは積み増し。最初から全部作らない。

## 15. NOT A HOTEL 版への寄せ方（具体提案）

ベースエンジンはそのまま流用し、**「誰が・何のために・どんな英語を」**を入れ替えます。

### 15.1 ターゲットと利用シーン

- **対象**: NOT A HOTEL の現場スタッフ（ヴィラホスト / コンシェルジュ / フロント / ハウスキーピング / F&B / 運営）。
- **目的**: 海外ゲストを上質におもてなしするための実戦英語。
- **利用シーン**: 出勤前の数分、移動中、シフトの合間。**短時間・高頻度**前提のままで完璧に合う。

### 15.2 プロフィールの読み替え

- 「家族メンバー」→「**スタッフ / 拠点（プロパティ）別チーム**」。
- `level` は英検目安のまま流用可（社内の英語ティアに読み替え）。
- 拠点別・職種別に `forProfileIds` 相当でタイルや教材を出し分け（例: コンシェルジュにはコンシェルジュ会話、ハウスキーピングには客室対応英語）。

### 15.3 コンテンツの寄せ方（ここが本番）

| ベース機能 | NOT A HOTEL 版の中身 |
|---|---|
| 単語学習 | ホスピタリティ語彙（amenity, turndown, late checkout, sommelier, itinerary…） |
| 定型構文（型 × 3 例） | **ゲスト対応の鉄板フレーズ**を型で。例: "Would you like me to …?" / "Allow me to …" / "I'll make sure that …" を 3 シーンずつ |
| シャドーイング単文 | チェックイン / ご案内 / リクエスト対応 / トラブル対応 / 見送り の定型 |
| 長文（passages） | **ブランドストーリー / 各プロパティ紹介 / おもてなし哲学 / 周辺アクティビティ案内**を英語で。スタッフがゲストに語れるように |
| リスニング理解 | ゲストの要望（早朝送迎・アレルギー・記念日…）を聞き取って正しく対応 |
| リーディング理解 | 予約確認メール / ハウスルール / 周辺ガイドの読解 |
| 1 分解説 | 接客で頻出の文法・敬意表現（婉曲・丁寧な依頼・クッション言葉） |
| AI 翻訳 / AI 添削 | ゲストへの英文メッセージ作成支援・添削 |
| レベル診断 | 入社時・配属時の英語アセスメント |

### 15.4 ホスピタリティ特化で足したい新機能（任意）

- **シーン別ロールプレイ**: 「チェックイン」「コンシェルジュ相談」「クレーム対応」をシナリオで。
- **発音重視モード**: ゲスト名・地名・料理名など固有名詞の発音練習。
- **「今日のおもてなしフレーズ」**: 1 日 1 つ、現場で即使える上質な一言。
- **拠点ローカル情報の英語化**: 各プロパティ周辺のレストラン・体験を英語で説明できるカード。

### 15.5 デザイン

- 配色・フォントを **NOT A HOTEL のブランドガイド**に差し替え（`C` とフォント定数を入れ替えるだけ）。
- トーンは「静けさ・余白・上質」。現行アプリの方向性と相性が良い。

### 15.6 運用・配布

- 社内配布なら GitHub Pages（限定公開）or 社内ホスティングに `.html` を置くだけ。
- スタッフは URL を開いて「ホーム画面に追加」→ アプリとして常駐。
- 進捗をチームで見たい場合のみ Supabase を有効化。
- AI 機能を全社展開するなら、API キーは**サーバー側プロキシ**で保持（端末直叩きにしない）。

---

## 16. 既知のトレードオフ（正直に共有）

- **単一 HTML が巨大**（約 1.6MB / 3 万行）。エディタは重いが、配布・運用は最強に楽。コンテンツが更に増えるなら、データ部分を別 JSON に切り出す手もある。
- **同期は last-write-wins**。同一人物が 2 端末で同時編集すると一方が消える。個人/小チームでは許容範囲。
- **AI キーのブラウザ管理**は個人利用前提。広域配布ではプロキシ必須。
- **音声は端末依存**。高品質ボイス未導入の端末では平凡な音になる（導入手順の案内でカバー）。

---

## 17. まとめ（仲間に渡すときの要点）

- **核は「単一 HTML・オフラインファースト・ゼロ摩擦」**。これさえ守れば中身は自由。
- **エンジン（レベル自動調整 / 今日のパス / 覚えた・間違い / 定型構文の 3 モード）は再利用**できる。
- **入れ替えるのは「誰が・何の英語を」だけ**。NOT A HOTEL 版なら、ホスピタリティ英語に総入れ替え。
- まずは単語＋瞬間英作文＋シャドーイングの 3 つで MVP を出し、走りながら積み増す。
